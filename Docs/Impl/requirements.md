# 仕様書：Viewport to Texture Baker

## 1. 概要 (Overview)

選択した複数のメッシュオブジェクトにおいて、割り当てられている各マテリアルの「ビューポート表示」設定（Color, Metallic, Roughness）を抽出し、重ならないUVマップを元にそれぞれ1枚のテクスチャとして結合・ベイク出力する拡張機能。

## 2. 動作環境・ターゲット (Environment)

* **対象ソフトウェア**: Blender 5.x.x 以降
* **配布形式**: Blender Extensions API (Blender 4.2以降の標準パッケージ形式に対応)
* **前提条件**: 対象オブジェクトのUVマップが展開済みであり、かつUVアイランドが重なっていないこと。

## 3. 機能要件 (Functional Requirements)

1. **対象オブジェクトの複数選択対応**:
* 選択中のすべてのMeshオブジェクトを対象とし、シームレスに処理を実行する。


2. **ビューポート値の抽出**:
* 各ポリゴンに割り当てられたマテリアルの以下プロパティを取得する。
* Base Color (`material.diffuse_color` のRGB値)
* Metallic (`material.metallic`)
* Roughness (`material.roughness`)




3. **パディング（補間）機能**:
* UVの境界線ギリギリで出力された際の問題を防ぐため、UVアイランドの境界から指定ピクセル分、色を外側に拡張して塗り足す（Bleed / Margin処理）。


4. **テクスチャ出力**:
* 抽出した3つの要素を、それぞれ独立した画像ファイルとして出力する。
* 解像度の指定が可能（例: 1024, 2048, 4096など）。



## 4. UI / UX 設計 (User Interface & Experience)

ユーザーの「右クリックで簡単に」という要望を満たすため、アクセス性と手軽さを最優先します。

* **アクセスポイント**: 3Dビューポート上でオブジェクトを選択し、右クリックで表示される「オブジェクトコンテキストメニュー (`VIEW3D_MT_object_context_menu`)」の下部に配置。
* **メニュー名**: `Export Viewport to Textures` (または「ビューポート設定をテクスチャにベイク」)
* **操作フロー**:
1. 統合したい複数のメッシュを選択。
2. 右クリックメニューから本機能を実行。
3. 実行後、画面左下の「オペレーターを調整 (Adjust Last Operation)」パネルが開き、以下の設定を事後調整できる（調整のたびに再計算・上書き保存）。
* **Resolution (解像度)**: 1024, 2048, 4096 など
* **Margin (パディング幅)**: 16px など（デフォルト値を設定）
* **Output Path (出力先)**: デフォルトはDocuments/ViewportToTextureBaker/プロジェクト名/ とする
* **テクスチャパッキング**: デフォルトはオフ。オンにした場合はBaseColorとRoughness/Metallicを1枚のテクスチャにまとめる。チャンネルは指定可能にする。





## 5. 内部処理フロー・実装アプローチ (Implementation Logic)

Python API (`bpy`) を使用し、内部的にBlenderの堅牢なBake機能をハックする形で実装します。

1. **環境状態の保存**: 現在のレンダリングエンジン（Eevee等）を記憶し、バックグラウンドで一時的に `Cycles` に切り替えます（ベイク機能はCyclesに依存するため）。
2. **一時マテリアル・ノードの自動構築**:
* スクリプト内で、各マテリアルに対して一時的なノードツリーを構築します。
* ビューポートの設定値を、ライティングの影響を受けない **`Emission (放射)`** ノードに接続します。


3. **画像データの生成とベイク (`bpy.ops.object.bake`)**:
* 指定解像度の空画像を生成し、Bake Typeを `Emit` に設定。
* Bake設定の `Margin` パラメータを利用してパディング処理を適用。
* Color、Metallic、Roughness 用のノード接続を内部で自動的に切り替えながら、計3回のベイクを連続実行します。


4. **ファイルの保存とクリーンアップ**:
* ベイクされた3枚の画像（PNGやTarga等）を指定パスに保存。
* 処理完了後、生成した一時ノードやマテリアルを削除し、レンダラー等の設定を元の状態に復元します（ユーザーの目には変更が見えないようにします）。

### その他
**自動ファイル命名規則**: 出力されるファイル名は `[Blendファイル名]_BaseColor.png` のような規則で自動生成する。同名のファイルが存在する場合は上書きするか確認し、しない場合は連番を付与して別名保存する。
